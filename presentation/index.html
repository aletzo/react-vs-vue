at this point we'd like to talk a bit about the differences of designing apps with react and vue

one major selling point for react is that it follows the functional paradigm

for the uninitiated ones, the definition alone of functional is a topic for a whole presentation

but one key attributes of functional programming is the development of code that has no side-effects

side-effects are the unintentional results that happen the code is executing

and the lack of side-effects is very desirable because it makes the code more predictable, so less buggy

along with one-way flow that we omitted to provide an example

maybe we can show some example at the end of the presentation, if we have enough time

it makes the code even more predictable, which again it means less room for human error, so less bugs

one more selling point for react is the freedom of choice when it comes to complementary libraries

the react community is very big and other the last 7 years has created many alternative libraries that solve the same problems

and this freedom of choice was by design in react

which might be good for some teams and might be bad for some other teams

freedom always comes at a cost

one other thing that react is known for is breaking changes on major version updates

and this is something that all teams should account enough time for,

because lagging behind in versions might result to an unsecure codebase

or a codebase that can't be updated or updated at a really slow, uncompetitive pace

this issue with breaking changes is something that vue doesn't have so far, but might change in the future

finally, the complexity for starting a project in react is a bit harder because of JSX that needs some additional tool to convert it to javascript

as we said earlier, the answer to our question for which library is better is

"it depends"

so now depending on the use-case of the team or the company or the project

here's where react is better

architecture: react is designed to make use of functional design patterns, resulting to less error-prone code

on top of that, seeing that the programming world shifts slowly to more functional design patterns or to more functional languages

it seems a good idea to invest on react, for a team that wants to stay current with the programming trends

talent pool: as we said earlier, the community of react seems to be significantly larger,

so if a team wants access to a larger talent pool, react is the way to go

native mobile apps: react native is a mature technology, while vue native is relatively new,

so if a team wants to build native mobile apps using javascript, react seems to be the safe choice

and what about vue?

legacy codebase: from our own experience and from the preceding examples, vue the obvious choice for modernizing an old codebase

tech stack limitations: it might occur that setting up a nodejs environment is out of question, so

the necessary tools for converting JSX to javascript might not be available, so react can't be used

performance: on some occasions the apparent performance advantage of vue over react might be a deciding factor, so in these cases vue will win
